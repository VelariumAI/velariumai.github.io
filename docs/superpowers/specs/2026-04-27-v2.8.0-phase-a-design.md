## 1. Goal

Add three new infrastructure modules to VCSE without modifying any existing code paths:

1. **GlobalClaimDedup** — cross-run duplicate detection with full provenance preservation
2. **PackIndex** — claim-level metadata index across all packs, separate from PackRegistry
3. **PackLifecycleManager** — enforced lifecycle transitions (candidate → certified → frozen → archived)

Plus Phase A CLI commands: `vcse pack index/list/info/freeze/archive/create`.

**Core principle:** Additive only. Zero changes to ConflictResolver, PackRegistry, TrustPromoter, LedgerStore, or CakePackUpdater logic (except one new guard call).

---

## 2. Non-Goals (Phase A)

- No general_world pack (Phase B)
- No query normalizer (Phase C)
- No CAKE incremental mode (Phase C)

## 3. Component: GlobalClaimDedup (`knowledge/dedup.py`)

### Purpose

Prevent duplicate claims from being written to a pack across multiple CAKE acquisition runs. ConflictResolver handles within-batch semantic dedup; GlobalClaimDedup handles cross-run exact-key dedup against on-disk state.

### Responsibility Boundary

| System | Scope |
|---|---|
| `ConflictResolver` | Dedup within a single in-memory batch of claims |
| `GlobalClaimDedup` | Dedup new claims against existing on-disk claims in a target pack |

### Data Model

```python
@dataclass
class DedupResult:
    unique_claims: list[KnowledgeClaim]
    # claims whose key does not appear in existing_claims

    duplicates_detected: list[KnowledgeClaim]
    # claims whose key already exists in existing_claims

    merged_provenance_map: dict[str, list[KnowledgeProvenance]]
    # key = claim.key (subject|relation|object)
    # value = ALL known provenance for that key:
    #   [existing_claim.provenance] + [duplicate.provenance for each dup with same key]
    # This preserves full audit trail even when claim body is not re-added
```

### Interface

```python
def deduplicate_claims(
    existing_claims: list[KnowledgeClaim],
    new_claims: list[KnowledgeClaim],
) -> DedupResult
```

### Rules

- Key = `claim.key` = `"|".join([subject, relation, object])` — exact string match only
- No fuzzy matching, no semantic comparison, no ML
- Duplicate claim body is NOT added to pack again
- Provenance from BOTH existing claim AND every duplicate incoming claim is recorded in `merged_provenance_map`
- `merged_provenance_map[key]` = `[existing.provenance] + [dup.provenance for each dup]`
- Deterministic: same inputs always produce same output (key ordering via sorted iteration)
- Does not write to disk — pure function, returns `DedupResult`

### Integration Point

`CakePackUpdater.update()` calls `deduplicate_claims(existing_claims, normalized_new_claims)` before any disk write. Only `unique_claims` are written. `merged_provenance_map` values are appended to `provenance.jsonl` for keys that had duplicates.

## 4. Component: PackIndex (`packs/index.py`)

### Purpose

Claim-level metadata index for all packs. Answers questions like: "how many certified claims does pack X have?" and "which packs contain claims from source Y?". Complements `registry.json` (which tracks installation).

### Storage

`~/.vcse/packs/index.json` — JSON object keyed by `pack_id@version`.

### Entry Schema

```json
{
  "wikidata_capitals@1.0.0": {
    "pack_id": "wikidata_capitals",
    "version": "1.0.0",
    "domain": "general",
    "lifecycle_status": "candidate",
    "claim_count": 6,
    "certified_count": 0,
    "candidate_count": 6,
    "source_ids": ["wikidata_capitals"],
    "last_updated": "2026-04-27T12:00:00+00:00",
    "pack_path": "/home/user/.vcse/cake/packs/wikidata_capitals",
    "stale": false
  }
}
```

**Trust count rules (deterministic):**
- `trust_tier == "T5_CERTIFIED"` → counted in `certified_count`
- missing trust_tier or any other tier → counted in `candidate_count`
- `certified_count + candidate_count == claim_count` always

**Stale rules:**
- `"stale": true` means pack_path no longer exists on disk (set by `build_index()`)
- Stale entries are retained in index (not deleted) for audit
- Archived packs are NOT marked stale — stale is a path-existence condition only

### Interface

```python

    def __init__(self, index_path: Path | None = None) -> None:
        # default: ~/.vcse/packs/index.json

    def build_index(self, scan_dirs: list[Path]) -> None:
        # Scans all pack dirs in scan_dirs for pack.json + claims.jsonl
        # Rebuilds entries for found packs (idempotent)
        # Marks entries for packs whose path no longer exists as stale=True
        # Does NOT delete any existing entries

    def load_index(self) -> dict[str, dict]:
        # Loads and validates index.json
        # Raises PackIndexError("CORRUPTED_INDEX", ...) if malformed

    def update_entry(self, pack_path: Path) -> None:
        # Recomputes metadata for ONE pack at pack_path
        # Writes updated entry back to index.json
        # Key = pack_id@version from pack.json

    def list_packs(self, include_stale: bool = False) -> list[dict]:
        # Returns entries sorted by (pack_id, version)
        # include_stale=False: omits stale=True entries

    def get_pack_metadata(self, pack_id_version: str) -> dict:
        # pack_id_version format: "pack_id@version" or "pack_id" (returns latest by semver)
        # Raises PackIndexError("PACK_NOT_FOUND", ...) if not found

class PackIndexError(ValueError):
    def __init__(self, error_type: str, reason: str) -> None: ...
```

### Rules

- Keyed by `pack_id@version` — multiple versions of same pack coexist
- Append/update only: no silent delete of existing entries
- `build_index()` is idempotent
- `update_entry()` called by `CakePackUpdater.update()` after each successful pack write
- Validated on load — malformed index raises error rather than silently returning partial data

## 5. Component: PackLifecycleManager (`packs/lifecycle.py`)

### Purpose

Enforce valid lifecycle state transitions for packs. Prevent writes to frozen/archived packs. Provide `create_version()` to produce a mutable copy of a frozen pack.

### Lifecycle States

```
candidate  →  certified  →  frozen  →  archived
    ↓              ↓            ↓
 archived       archived     archived
    ↓
 (terminal)
```

`lifecycle_status` stored as string field in `pack.json`:
```json
{"lifecycle_status": "candidate"}
```

Missing field treated as `"candidate"`.

### Allowed Transitions (explicit table)

| From | To | Allowed |
|---|---|---|
| candidate | certified | ✅ |
| candidate | frozen | ✅ |
| candidate | archived | ✅ |
| certified | frozen | ✅ |
| certified | archived | ✅ |
| frozen | archived | ✅ |
| archived | any | ❌ |
| frozen | certified | ❌ |
| frozen | candidate | ❌ |
| certified | candidate | ❌ |

Any unlisted transition is disallowed.

### Interface

```python
class PackLifecycleManager:
    def get_status(self, pack_path: Path) -> str:
        # Reads lifecycle_status from pack.json. Missing = "candidate".

    def transition(self, pack_path: Path, new_status: str) -> None:
        # Validates transition via table above.
        # Raises PackLifecycleError("INVALID_TRANSITION", ...) on violation.
        # Writes new lifecycle_status to pack.json.

    def freeze_pack(self, pack_path: Path) -> None:
        # Calls transition(pack_path, "frozen")

    def archive_pack(self, pack_path: Path) -> None:
        # Calls transition(pack_path, "archived")

    def create_version(self, pack_path: Path, new_version: str) -> Path:
        # Validates SemVer format (X.Y.Z digits only).
        # Raises PackLifecycleError("INVALID_SEMVER", ...) if malformed.
        # Determines new pack dir: same parent as pack_path, name = pack_id@vX.Y.Z
        # Raises PackLifecycleError("VERSION_EXISTS", ...) if target dir already exists.
        # Copies pack_path directory tree to new dir (shutil.copytree).
        # Sets new pack's lifecycle_status = "candidate" (fresh version starts writable).
        # Returns new pack Path.
        # Original pack is UNCHANGED.

    def assert_writable(self, pack_path: Path) -> None:
        # Raises PackLifecycleError("PACK_FROZEN", ...) if status == "frozen".
        # Raises PackLifecycleError("PACK_ARCHIVED", ...) if status == "archived".
        # No-op if candidate or certified.

class PackLifecycleError(PackError):
    def __init__(self, error_type: str, reason: str) -> None: ...
```

### CakePackUpdater Integration

Single new line at start of `CakePackUpdater.update()`:
```python
PackLifecycleManager().assert_writable(pack_path)
```
Raises `PackLifecycleError` before any disk write if frozen or archived. No other logic changes.

## 6. Phase A CLI Commands

All added to `src/vcse/cli.py` under `pack` subparser (existing subparser gets new subcommands).

### Command Specifications

**`vcse pack index [--dirs <path>...]`**
- Rebuilds `~/.vcse/packs/index.json` deterministically from disk scan
- Marks entries whose pack_path no longer exists as `"stale": true` — does NOT delete them
- Validates index before overwrite (write new then replace atomically)
- Output: `{"status": "INDEXED", "packs_found": N, "stale": M}`

**`vcse pack list [--stale] [--json]`**
- Reads from `index.json` only — does NOT scan disk
- Sorts by `(pack_id, version)` ascending
- `--stale`: include stale entries (default: omit)
- Always includes `lifecycle_status` in each entry
- Output: JSON array of pack summaries

**`vcse pack info <pack_id>[@<version>] [--json]`**
- `pack_id@version`: exact lookup
- `pack_id` alone: returns latest version (highest SemVer)
- Required output fields: `version`, `lifecycle_status`, `claim_count`, `certified_count`, `candidate_count`, `source_ids`, `pack_path`, `last_updated`
- Raises structured error `PACK_NOT_FOUND` if not in index
- Read-only: no side effects

**`vcse pack freeze <pack_id>[@<version>] --path <pack_path>`**
- Enforces lifecycle transition table: must be `candidate` or `certified` → `frozen`
- Raises `INVALID_TRANSITION` if already frozen or archived
- Writes `lifecycle_status = "frozen"` to `pack.json` only
- Updates index entry via `PackIndex.update_entry()`

**`vcse pack archive <pack_id>[@<version>] --path <pack_path>`**
- Sets `lifecycle_status = "archived"` — does NOT delete directory
- Pack remains visible in index (omitted from `list` unless `--stale` or `--archived` flag)
- Raises `INVALID_TRANSITION` if already archived

**`vcse pack create <name> --path <dir>`**
- Creates `<dir>/<name>/` directory
- Initializes: `pack.json` (with `lifecycle_status: "candidate"`, `claim_count: 0`), empty `claims.jsonl`, empty `provenance.jsonl`
- Raises `PACK_EXISTS` if directory already exists
- Adds entry to index via `PackIndex.update_entry()`

### Global CLI Rules

- All commands support `--json` flag for machine-readable output
- No Python tracebacks on user errors — structured error with `error_type` field to stderr
- `list` and `info` are strictly read-only
- `freeze`, `archive`, `create` are explicit mutations only
- Version: `pack_id@version` explicit OR `pack_id` → latest by SemVer (deterministic)

## 11. Acceptance Criteria

- `deduplicate_claims()` is deterministic: same inputs → same output
- `merged_provenance_map` contains provenance from both existing claim AND all duplicates
- `PackIndex` keyed by `pack_id@version`; multiple versions coexist
- Stale entries retained (not deleted) when pack_path missing
- All transition table entries enforced; invalid transition raises `PackLifecycleError`
- `create_version()`: validates SemVer, fails if target exists, copies, preserves original
- `CakePackUpdater.update()` raises before disk write if pack frozen/archived
- Phase A tests: 3 new test files, all pass
- No regression in existing 491 tests
- `false_verified_count: 0`
