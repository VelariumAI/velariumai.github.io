# Versioning

VCSE uses SemVer during prototype development.

- `0.1.0`: hardened world-state memory
- `0.2.0`: formal transition model
- `0.3.0`: verifier stack
- `0.4.0`: stable bounded search
- `0.5.0`: symbolic proposers
- `0.6.0`: deterministic renderer
- `0.7.0`: CLI demos and JSON input
- `0.9.0`: benchmark harness and stabilization
- `1.0.0`: stable VCSE release

Every tagged version must pass tests, run the CLI, and keep the engine CPU-only.
Patch releases are reserved for fixes and refactors.
