# COMPREHENSIVE AGENT DIRECTIVE
## Thorough Repository Codebase Analysis Protocol (v2.1 - Exhaustive Edition)

**Directive ID:** AGENT-DIR-RA-001  
**Effective Date:** April 27, 2026  
**Purpose:** This directive transforms the high-level instruction *"perform a thorough repo analysis of the code base"* into an **exhaustive, mandatory, step-by-step protocol**. Every agent MUST adhere to it without omission, deviation, or summarization shortcuts. The analysis must be **bottom-up, detail-oriented, evidence-based, and include precise Lines of Code (LOC) counts at every granularity level** (global, per-language, per-directory, per-file where feasible).

**Core Principles (Non-Negotiable):**
- **Exhaustiveness:** Analyze from the absolute filesystem root downward. No file, directory, config, or metric may be skipped.
- **Evidence Mandate:** Every finding must cite exact command output, file path + line numbers, or code excerpts. Never rely on assumptions.
- **LOC Precision:** LOC is the cornerstone metric. Provide **total lines**, **source LOC** (excluding blanks + comments), **comment LOC**, **blank LOC** — broken down by language, directory, and component.
- **Tool Usage:** Primary tools = bash (find, grep, wc, du, git, ls), Python (custom LOC counters, JSON/YAML parsers), and any pre-installed analyzers. No external internet calls.
- **Output Standard:** Structured Markdown report saved to `/home/workdir/artifacts/thorough_repo_analysis_[REPO_NAME]_[TIMESTAMP].md`. Include tables, code blocks, and phase completion markers.
- **Adaptivity:** Detect stack (Python, JS/TS, Go, Java, Rust, etc.) dynamically and apply language-specific deep dives.
- **Safety:** Never execute destructive commands. Redact all secrets in reports. Flag but do not expose credentials.

**Pre-Execution Checklist (Run Before Step 1):**
- Confirm `pwd` == repository root.
- Verify write access to `/home/workdir/artifacts/`.
- Run `date` and record analysis start timestamp.
- If `.git` absent, note "Non-Git repository" and proceed with filesystem-only analysis.

**Robustness & Edge-Case Safeguards (v2.1 Enhancements — Mandatory):**
- **Massive File Guard:** Never run unbounded `grep -r`, `cat`, or text searches on files >2MB. Always append size filters: `find . -type f -size -2M ...` or `timeout 30s grep ...`. This prevents hangs on binaries, large SQLite DBs, or minified .map files.
- **Encoding Resilience:** The LOC script below uses multi-encoding fallback (utf-8 → utf-16 → windows-1252 → latin-1). If counts appear suspiciously low on legacy files, manually inspect with `file -i [filepath]` and re-count.
- **Timeout Guardrails:** For any potentially long-running command (deep recursive searches, dependency parsing, full-repo grep), prefix with `timeout 60s` (or shorter). Example: `timeout 30s grep -r "TODO" --include="*.py" .`
- **Output Redirection Mandate:** All heavy commands MUST redirect to files (`> /tmp/phaseX_output.txt`) and use `head -100`, `tail -50`, or `grep` to feed summaries back to the agent. Never paste >500 lines of raw terminal output.

## PHASE 3: EXHAUSTIVE LINES OF CODE (LOC) ANALYSIS — CORE METRIC

**Objective:** Deliver **precise, multi-dimensional LOC counts**. This is the most critical phase.

**Mandatory LOC Protocol (No Shortcuts):**
Since `cloc` is typically unavailable, **MUST** use custom Python counter script for accuracy.

**Step 3.1: Deploy Universal LOC Counter Script**
Create and execute the following Python script (save as `/tmp/loc_counter.py`):

```python
#!/usr/bin/env python3
import os
import sys
from collections import defaultdict
import re

# Language comment patterns (extend as needed)
COMMENT_PATTERNS = {
    'py': (r'^\s*#', r'^\s*"""', r'^\s*\'\'\''),
    'js': (r'^\s*//', r'^\s*/\*', r'^\s*\*'),
    'ts': (r'^\s*//', r'^\s*/\*'),
    'go': (r'^\s*//', r'^\s*/\*'),
    'java': (r'^\s*//', r'^\s*/\*'),
    'c': (r'^\s*//', r'^\s*/\*'),
    'cpp': (r'^\s*//', r'^\s*/\*'),
    'rs': (r'^\s*//', r'^\s*/\*'),
    'rb': (r'^\s*#'),
    'php': (r'^\s*//', r'^\s*#', r'^\s*/\*'),
    'sh': (r'^\s*#'),
    'yaml': (r'^\s*#'),
    'yml': (r'^\s*#'),
    'json': (),  # no comments
    'toml': (r'^\s*#'),
    'md': (),    # treat as docs, not code
}

def is_comment(line, lang):
    patterns = COMMENT_PATTERNS.get(lang, ())
    for p in patterns:
        if re.match(p, line):
            return True
    return False

def count_loc(filepath):
    """
    v2.1 Enhanced: Multi-encoding fallback to handle UTF-16, Windows-1252, Latin-1, etc.
    Falls back gracefully and logs encoding used for suspicious files.
    """
    encodings_to_try = ['utf-8', 'utf-16', 'windows-1252', 'latin-1', 'cp1252']
    lines = None
    used_encoding = None
    for enc in encodings_to_try:
        try:
            with open(filepath, 'r', encoding=enc, errors='strict') as f:
                lines = f.readlines()
                used_encoding = enc
                break
        except (UnicodeDecodeError, UnicodeError):
            continue
    if lines is None:
        # Final fallback with ignore (rare)
        try:
            with open(filepath, 'r', encoding='utf-8', errors='ignore') as f:
                lines = f.readlines()
                used_encoding = 'utf-8-ignore'
        except:
            return 0, 0, 0, 0, 'error'
    total = len(lines)
    blank = sum(1 for l in lines if l.strip() == '')
    lang = os.path.splitext(filepath)[1][1:].lower()
    if lang == '': lang = 'unknown'
    comments = sum(1 for l in lines if is_comment(l, lang))
    source = total - blank - comments
    # Optional: print suspicious encoding for manual review (comment out in prod if too verbose)
    if used_encoding and used_encoding != 'utf-8':
        print(f"  [ENC] {filepath}: used {used_encoding}")
    return total, source, comments, blank, lang

# Walk and aggregate
stats = defaultdict(lambda: {'files': 0, 'total': 0, 'source': 0, 'comments': 0, 'blank': 0})
dir_stats = defaultdict(lambda: {'files': 0, 'total': 0, 'source': 0})

root = '.'
for dirpath, dirnames, filenames in os.walk(root):
    # Skip common build/venv dirs
    if any(x in dirpath for x in ['.git', 'node_modules', '.venv', 'venv', 'target', 'build', 'dist', '__pycache__']):
        continue
    for fname in filenames:
        fpath = os.path.join(dirpath, fname)
        ext = os.path.splitext(fname)[1][1:].lower()
        if ext in ['py', 'js', 'ts', 'jsx', 'tsx', 'go', 'java', 'c', 'cpp', 'h', 'hpp', 'rs', 'rb', 'php', 'sh', 'bash', 'yaml', 'yml', 'toml', 'json', 'md', 'txt', 'rst']:
            t, s, c, b, lang = count_loc(fpath)
            stats[lang]['files'] += 1
            stats[lang]['total'] += t
            stats[lang]['source'] += s
            stats[lang]['comments'] += c
            stats[lang]['blank'] += b
            # Directory aggregation (top 2 levels)
            rel_dir = os.path.relpath(dirpath, root)
            top_dir = rel_dir.split(os.sep)[0] if rel_dir != '.' else 'root'
            dir_stats[top_dir]['files'] += 1
            dir_stats[top_dir]['total'] += t
            dir_stats[top_dir]['source'] += s

# Output grand totals
grand_total = sum(v['total'] for v in stats.values())
grand_source = sum(v['source'] for v in stats.values())
print("=== GLOBAL LOC SUMMARY ===")
print(f"Total Files Analyzed: {sum(v['files'] for v in stats.values())}")
print(f"Grand Total Lines: {grand_total}")
print(f"Source LOC (excl. blank+comment): {grand_source}")
print(f"Comment LOC: {sum(v['comments'] for v in stats.values())}")
print(f"Blank LOC: {sum(v['blank'] for v in stats.values())}")
print("\n=== BY LANGUAGE ===")
for lang in sorted(stats, key=lambda x: -stats[x]['source']):
    v = stats[lang]
    print(f"{lang:8} | Files: {v['files']:5} | Total: {v['total']:7} | Source: {v['source']:7} | Comments: {v['comments']:6} | Blank: {v['blank']:6}")

print("\n=== BY TOP-LEVEL DIRECTORY ===")
for d in sorted(dir_stats, key=lambda x: -dir_stats[x]['source']):
    v = dir_stats[d]
    print(f"{d:20} | Files: {v['files']:5} | Total: {v['total']:7} | Source: {v['source']:7}")
```

**Execution:**

## CHAT INTERFACE EXECUTION PROTOCOL (v2.1 — Critical for Token/Context Management)

### 1. The Execution Gap
The AI cannot directly execute bash/Python commands in most chat interfaces. **You (the human) must act as the bridge:**
- AI outputs the exact command(s) for the current phase.
- You copy, run in your local terminal (or sandbox), and paste **only the relevant stdout/stderr** back.
- AI then parses the output and proceeds.

**Rule:** Never let the AI "pretend" it ran commands. Demand real output.

### 2. Phase-by-Phase Pausing (Token Limit Protection)
Chat models have strict per-response token limits (~4k–8k). A full 12-phase report will explode the context.

**Mandatory Protocol:**
- After completing each phase (including the Phase X Completion Marker), the agent **MUST pause** and output:

### 3. Terminal Output Truncation Rules
Heavy commands (LOC script, full tree, recursive grep) can produce 10k+ lines.

**Mandatory Safeguards (already partially in v2.1 Robustness section):**
- All long commands **MUST** use redirection: `python3 /tmp/loc_counter.py > /tmp/loc_report.txt 2>&1`
- Feed the AI **only summaries**:
  - `head -80 /tmp/loc_report.txt` (global + language tables)
  - `grep -E "===|Source LOC|Total Files" /tmp/loc_report.txt`
  - For debt markers: `grep -c "TODO\|FIXME" /tmp/debt_scan.txt` instead of full list
- If output still too large: `wc -l /tmp/xxx.txt` + `head -30` + `tail -20`

**Golden Rule:** The agent should never receive >400 lines of raw terminal output in a single message. Summarize or section it.

### 4. Context Decay Prevention
